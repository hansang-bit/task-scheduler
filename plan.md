# 일정 관리 캘린더 앱 - 구현 계획서

## 개요
사내 작업자별 일정을 입력하고 관리할 수 있는 엑셀 시트 같은 캘린더 웹 앱.
GitHub Pages에 호스팅하여 팀원 누구나 접속 가능. Supabase(PostgreSQL)로 데이터 공유.

## 기술 스택
- **프론트엔드**: 순수 HTML5 Canvas API + Vanilla JavaScript (단일 index.html)
- **백엔드 DB**: Supabase (무료 PostgreSQL + REST API + Realtime)
- **호스팅**: GitHub Pages
- **오프라인 지원**: Supabase 미설정 시 localStorage fallback
- **외부 라이브러리**: Supabase JS SDK (esm.sh CDN)만 허용

## 파일 구조
```
/
├── index.html    # 전체 애플리케이션 (HTML + CSS + JS 단일 파일)
├── CLAUDE.md     # 프로젝트 규칙 + Supabase 설정 가이드
└── plan.md       # 이 파일
```

## 데이터 모델

### Supabase 테이블
| 테이블 | 컬럼 | 설명 |
|--------|-------|------|
| `workers` | id (PK), name, active | 작업자 목록 |
| `projects` | id (PK), name, active | 프로젝트 목록 |
| `tasks` | id (PK), worker_id (FK), project_id (FK), description, start_date, end_date, working_days | 일감/작업 |

### 로그인 상태
- `localStorage`에 현재 선택된 worker ID 저장
- 비밀번호 없음 — 이름 선택만으로 입장

## 페이지 구성

### SPA 라우팅 (Hash 기반)
| Hash | 페이지 | 접근 권한 |
|------|--------|-----------|
| `#login` | 로그인 | 전체 |
| `#worker` | 작업자 일정 입력 | 로그인 필요 |
| `#admin` | 관리자 페이지 | 조한상만 |

### 1. 로그인 페이지
- 드롭다운에서 이름 선택 → "입장" 버튼
- 조한상 선택 시 작업자 페이지 + 관리자 페이지 모두 접근 가능

### 2. 작업자 페이지
- **상단 네비게이션**: 앱명, 사용자 이름, 로그아웃, 관리자 링크(조한상만)
- **좌측 패널**: 내 작업 목록 테이블
  - 컬럼: 프로젝트, 일감, 시작일, 종료일, 근무일수, 수정/삭제
  - "+ 추가" 버튼 → 모달 다이얼로그
- **우측 패널**: Canvas 간트차트
  - 날짜별 컬럼, 작업 바 시각화
  - 주말/공휴일 배경색 구분
  - 오늘 날짜 빨간 점선 표시
  - 클릭하면 작업 수정 모달 열림

### 3. 관리자 페이지 (조한상 전용)
- **전체 일정 탭**: 모든 작업자 일정 조회
  - 작업자별/프로젝트별 필터 드롭다운
  - 통합 테이블 + 간트차트 (작업자 이름 고정 컬럼)
  - 작업 수정/삭제 가능
- **작업자 관리 탭**: 이름 추가/수정/비활성화/삭제
- **프로젝트 관리 탭**: 프로젝트명 추가/수정/비활성화/삭제

## 간트차트 (Canvas)
- `devicePixelRatio` 대응 (Retina 디스플레이)
- 컬럼 너비 32px/일, 행 높이 36px, 헤더 50px
- 렌더링 레이어 순서: 배경 그리드 → 헤더(월/일/요일) → 작업 바 → 오늘 표시선
- 프로젝트별 12색 팔레트 자동 배정
- 관리자 간트: 작업자 이름 왼쪽 고정 컬럼 (스크롤 시 따라가지 않음)

## 한국 공휴일 처리
- 2025-2026년 공휴일 하드코딩 (음력 기반 날짜는 알고리즘 계산 불가)
- 대체공휴일 포함
- `isNonWorkingDay(dateStr)`: 토/일 + 공휴일 체크
- `countWorkingDays(start, end)`: 시작~종료일 사이 근무일수 계산 (공휴일 제외)
- 날짜 입력 시 기본값은 접속 당일 날짜

## Supabase 설정 (사용자 필수 작업)
1. [supabase.com](https://supabase.com) 회원가입
2. New Project 생성 (Region: Seoul)
3. SQL Editor에서 테이블 생성 SQL 실행 (CLAUDE.md 참조)
4. Project Settings → API에서 URL + anon key 복사
5. `index.html`의 `SUPABASE_URL`, `SUPABASE_KEY` 교체
6. GitHub에 push → GitHub Pages로 배포
